## Agent 开发小厂面试常见题

1. 如何设计一个 Prompt？Prompt Engineering 有哪些技术？（5 次）
2. 什么是 AI Agent？谈谈你理解的 Agent，以及如何理解其推理、决策与自主规划执行的能力？（3 次）
3. 什么是 Function Call？它的实现流程是什么？一般应用在哪些场景？（4 次）
4. 文本向量化的目的是什么？使用的向量模型、向量存储工具分别是什么？（3 次）
5. RAG 的概念是什么？其具体实现流程（包括 chunk 划分、向量搜索、重排序等环节）是怎样的？（3 次）
6. LangChain 的核心功能是什么？基于 LangChain 会调用哪些类型的模型？（2 次）
7. 大模型的调用方式是什么？本地部署大模型会用到哪些框架？（1 次）
8. 大模型 Server 是如何部署的？提供了哪些接口？是否支持流式输出？（1 次）
9. ReAct 的实现原理是什么？其 Prompt 设计和工程落地分别有哪些要点？（1 次）
10. 对 AI 编程、开源项目、Vibe Coding 分别有怎样的理解和看法？（3 次）

## 美团 Agent 开发（智能客服方向）二面

**项目背景与动机**

1. 能细节介绍一下你最近的多记忆健康问诊项目吗？（解决了什么问题、你担任什么角色、遇到什么难题）
2. 这个项目是实习中做的还是自己开发的？
3. 什么触动你去做这个项目？切入的初衷是什么？

**记忆系统设计**

1. 用一个例子解释一下你的 Memory 是怎么做的，以及怎么插入上下文的，怎么考虑上下文长度问题的？
2. 你结构化信息和文本信息之间如何区分？会不会重复？
3. 记忆冲突怎么解决？比如用户前后说了不同的过敏信息？
4. 记忆是怎么写入上下文的？上下文从头到尾结构是怎样的？
5. 上下文结构是不是：System Prompt + Memory + Tools Result + Conversation 这四个环节？
6. 上下文超长了怎么办？有没有考虑过上下文压缩？
7. 压缩过程中会丢失工具调用历史，导致模型重复调用工具，这个问题怎么解决？

**RAG 与检索**

1. 为什么用 RAG 去检索 Memory？为什么用向量这种方式？
2. Memory 的检索是分层次的吗？具体怎么设计的？
3. 检索中遇到了什么痛点？

**可观测性 / Trace**

1. 整个框架是很依赖埋点的，你觉得怎样的结构才能更好地追踪整个 Trace？
2. 日志怎么结构化？为什么要这么结构化？
3. 跨 Session 是否需要记录日志系统？
