# MySQL 概述

## 相关概念

- 数据库
  - 顾名思义就是存储数据的仓库，而且是**有组织的存储**。DataBase(**DB**)。而MySQL是数据库的一种，属于其中的**关系型数据库**
- 数据库管理系统
  - 操纵和管理数据库的大型软件。DataBaseManagementSystem(**DBMS**)
- SQL
  - 操作**关系型数据库**的编程语言，定义了一套操作关系数据库的**统一标准**。

**SQL ---> DBMS ---> DB**

## 关系型数据库

建立在**关系模型**基础上，由多张相互连接的**二维表**组成的数据库

特点：

1. 使用表存储数据，格式统一，便于维护
2. 使用SQL语言操作，标准统一，使用方便

## SQL语句

### SQL通用语法

#### SQL语句的特点

- sql语句可以单行或多行书写，以**分号结尾**
- sql语句可以使用空格/缩进来增强语句的可读性
- MySQL数据库的sql语句**不区分大小写**，关键字建议大写
- **注释**
  - 单行注释：`#` 这是sql语句特有的注释方式
  - 多行注释：`/* */`

#### SQL语句的分类

| 分类 |            全称            |                            说明                            |
| :--: | :------------------------: | :--------------------------------------------------------: |
| DDL  |  Data Definition Language  |  数据**定义**语言，用来定义数据库对象（数据库、表、字段）  |
| DML  | Data Manipulation Language |     数据**操作**语言，用来对数据库表中的数据进行增删改     |
| DQL  |    Data Query Language     |        数据**查询**语言，用来查询数据库中的表的记录        |
| DCL  |   Data Control Language    | 数据**控制**语言，用来创建数据库对象、控制数据库的访问权限 |

### DDL

#### DDL-数据库操作

##### 1.查询

- 查询所有数据库

```sql
SHOW DATABASES;
```

- 查询当前数据库

```sql
SELECT DATABASE();
```

##### 2.创建

```sql
CREATE DATABASE [IF NOT EXISTS] 数据库名 [DEFAULT CHARSET 字符集] [COLLATE 排序规则];
-- [IF NOT EXISTS]的作用是：如果这个数据库名称不存在则创建，如果存在则不创建
-- [DEFAULT CHARSET 字符集]是指定当前数据库所使用的字符集
-- [COLLATE 排序规则]则是指定排序规则
-- 在执行SQL语句时，是不写中括号的，这里只是为了方便区分才使用中括号
-- []所框起来的，都是可以省略的，省略不写的话都是执行数据库默认的规则
```

##### 3.删除

```sql
DROP DATABASE [IF EXISTS] 数据库名;
```

##### 4.使用

```sql
USE 数据库名;
```

#### DDL-表操作

##### 1.查询

- 查询当前数据库所有表

```sql
SHOW TABLES;
```

- 查询表结构

```sql
DESC 表的名字;
```

- 查询指定表的建表语句

```sql
SHOW CREATE TABLE 表名;
-- 这种查询方法比desc得到的信息要更为详细
```

##### 2.创建

```sql
CREATE TABLE 表名(
    字段1 字段1的类型 [COMMENT 字段1注释],
    ....
    字段n 字段n的类型 [COMMENT 字段n注释]
) [COMMENT 表注释];
-- 中括号里的东西也是可选的
```

##### 3.修改

- 添加字段

```sql
ALTER TABLE 表名 ADD 字段名 类型(长度) [COMMENT 注释] [约束];
```

- 修改表名

```sql
ALTER TABLE 表名 RENAME TO 新表名;
```

- 修改字段类型

```sql
ALTER TABLE 表名 MODIFY 字段名 新数据类型(长度);
```

- 修改字段名和字段类型

```sql
ALTER TABLE 表名 CHANGE 旧字段名 新字段名 类型(长度) [COMMENT 注释] [约束];
```

- 直接删除表

```sql
DROP TABLE [IF EXISTS] 表名;
-- 如果这个表名称存在则删除，如果不存在则不执行
```

- 删除指定表，并重新创建该表（使用较少）

```sql
TRUNCATE TABLE 表名;
```

- 删除字段

```sql
ALTER TABLE 表名 DROP 字段名;
```

#### DDL-数据类型

MySQL中的数据类型有很多，主要分为三类：数值类型、字符串类型、日期时间类型

##### 1.数值类型

|   类型    | 大小  |    有符号(SIGNED)范围    | 无符号(UNSIGNED)范围 |     描述     |
| :-------: | :---: | :----------------------: | :------------------: | :----------: |
|  TINYINT  | 1byte |        -128 - 127        |       0 - 255        | 非常小的整数 |
| SMALLINT  | 2byte |      -32768 - 32767      |      0 - 65535       |  稍小整数值  |
| MEDIUMINT | 3byte |    -8388608 - 8388607    |     0 - 16777215     |   小整数值   |
|    INT    | 4byte | -2147483648 - 2147493647 |    0 - 4294967295    |    整数值    |
|  BIGINT   | 8byte |      -2^63 - 2^63-1      |      0 - 2^64-1      |   大整数值   |
|   FLOAT   | 4byte |            -             |          -           | 单精度浮点数 |
|  DOUBLE   | 8byte |            -             |          -           | 双精度浮点数 |

##### 2.字符串类型

|    类型    |         大小         |                             描述                             |
| :--------: | :------------------: | :----------------------------------------------------------: |
|    CHAR    |    0 - 255 bytes     |    **定长**字符串(为达到额定的长度时会用空格补齐，性能好)    |
|  VARCHAR   |   0 - 65535 bytes    | **变长**字符串(只要未超过额定长度，就会根据字符长度决定串的长度，相对来说性能较差) |
|  TINYBLOB  |    0 - 255 bytes     |                 不超过255个字符的二进制数据                  |
|  TINYTEXT  |    0 - 255 bytes     |                         短文本字符串                         |
|    BLOB    |   0 - 65535 bytes    |                    二进制形式的长文本数据                    |
|    TEXT    |   0 - 65535 bytes    |                          长文本数据                          |
| MEDIUMBLOB |  0 - 16777215 bytes  |                 二进制形式的中等长度文本数据                 |
| MEDIUMTEXT |  0 - 16777215 bytes  |                       中等长度文本数据                       |
|  LONGBLOB  | 0 - 4294967295 bytes |                  二进制形式的极大文本数据                    |
|  LONGTEXT  | 0 - 4294967295 bytes |                         极大文本数据                         |

##### 3.日期时间类型

|   类型    |  大小   |                    范围                    |        格式         |       描述       |
| :-------: | :-----: | :----------------------------------------: | :-----------------: | :--------------: |
|   DATE    | 3 bytes |          1000-01-01 至 9999-12-31          |     YYYY-MM-DD      |      日期值      |
|   TIME    | 3 bytes |          -838:59:59 至 832:59:59           |      HH:MM:SS       | 时间值或持续时间 |
|   YEAR    | 1 bytes |                1901 至 2155                |        YYYY         |      年份值      |
| DATETIME  | 8 bytes | 1000-01-01 00:00:00 至 9999-12-31 23:59:59 | YYYY-MM-DD HH:MM:SS | 混合日期和时间值 |
| TIMESTAMP | 4 bytes | 1970-01-01 00:00:01 至 2038-01-19 03:14:07 | YYYY-MM-DD HH:MM:SS | 混合日期和时间值 |

### DML

#### DML-添加数据(INSERT)

##### 1.给指定字段添加数据

```sql
INSERT INTO 表名(字段名1, 字段名2, ...) VALUES(值1, 值2, ...);
-- 示例:
INSERT INTO user(id, name, age, gender, date) VALUES (1, 'lxxx', '20', '男', '2020-4-1');
```

##### 2.给所有字段添加数据

```sql
INSERT INTO 表名 VALUES(值1, 值2, ...);
-- 不指定字段名,就默认是给所有字段添加数据
```

##### 3.批量添加数据

```sql
INSERT INTO 表名(字段名1, 字段名2, ...) VALUES(值1, 值2, ...), (值1, 值2, ...), ...;
-- 批量即是指定多个值
```

**注意：**

- 插入数据时，指定字段顺序需要与值的顺序是**一一对应**的
- **字符串**和**日期型数据**应该包含在**引号**中
- 插入的数据大小应该在字段的**数据类型的规定范围内**

#### DML-修改数据(UPDATE)

```sql
UPDATE 表名 SET 字段名1 = 值1, 字段名2 = 值2, ... [WHERE 条件];
-- 示例：
-- 这个sql语句作用为将id为1的字段的name字段的值改为了指定值
UPDATE user SET name = '小小白' WHERE id = 1;
```

**注意：**

- 修改语句的条件可以忽略，如果忽略的话，则会默认修改整张表的数据
- 在忽略where条件时，图形化界面工具DataGrip会发出警告，点击Execute即可

#### DML-删除数据(DELETE)

```sql
DELETE FROM 表名 [WHERE 条件];
```

**注意**：

- DELETE 语句的条件同样可以忽略，如果**忽略**，则**会删除整张表的数据**
- DELETE语句**不能删除某个字段的某一个值**(如果有这种需求可以使用**UPDATE**,将想要删除的那个字段的值**设为null**即可)

```sql
-- 如果想要将id为1的date值删除就可以使用这行sql语句
UPDATE user SET date = null WHERE id = 1;
```
