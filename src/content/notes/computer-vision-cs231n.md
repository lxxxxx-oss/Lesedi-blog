# 计算机视觉 — CS231n 笔记

图像分类的任务就是对于一个给定的图像，预测它属于哪个分类标签（或者给出属于一系列不同标签的可能性）。

## Nearest Neighbor 分类器

作为课程介绍的第一个方法，这个分类器和卷积神经网络没有任何关系，实际中也很少使用，但通过实现它，可以使我们对于解决图像分类问题的方法有个基本的认识。

### L1 距离

**计算机是怎么比较两张图片的呢？** 在本例中，就是比较 32x32x3 的像素块。最简单的方法就是逐个像素比较，最后将差异值加起来，差异值越小就代表两张图片越相似，**如果差异值为零，就代表两张是一样的**。这个差异值就是 **L1 距离**，也被叫做曼哈顿距离。

![L1距离](/notes-images/L1距离.webp)

### L2 距离

**计算向量间的距离**的方法有很多种，另一个常用的就是 L2 距离，这就是我们最为熟知的**两点之间的距离公式**。

![L2距离](/notes-images/L2距离.png)

如图，对于 L1 距离而言，左图中正方形所有边上的点到坐标原点的 L1 距离相同，而对于 L2 距离而言，是右图中圆上所有的点到坐标原点的 L2 距离相同。可以发现，**L1 距离对于坐标轴旋转是十分敏感的**，如果对坐标系进行旋转之后，到原点距离相同的正方形也会跟着旋转，而 L2 距离对于坐标轴旋转却不敏感，即使坐标轴发生旋转，到原点距离相同的圆形也不会发生变化。

因此，**L1 距离比较适合坐标轴特征是有明确含义的情况**，比如员工分类的时候，特征向量不同元素对应着年龄、收入等有明确意义的特征。而 **L2 比较适合坐标轴（图片特征）没有明确含义的情况**。

通过设置不同的距离计算方式，可以将 NN（以及下面会提到的 KNN）泛化到不同领域，应用到不同的数据类型，比如 NLP 领域的文本格式。

## K-Nearest Neighbor 分类器

### KNN 的优势

由于在 Nearest Neighbor 中，只需要找最近的标签就可完成分类，但是这就**可能存在一个错误样本导致模型效果显著下降的问题**。如下图，图中的节点是训练集数据，节点的颜色表示训练集标签，而根据最近距离原则，可以将图片中不同区域划分为不同的标签。但是可以发现，在图片中间的绿色区域有一小块黄色区域，这显然是不合理的，实际上这个黄色标签可能是错误的，中间区域应该都是绿色标签，这就是 Nearest Neighbor 的局限性所导致的，所以 KNN 的出现就是必要的。

![KNN的优势](/notes-images/KNN的优势.png)

KNN 的思想也很简单：与其只找最相近的那一个图片的标签，我们干脆**就找出最相似的 K 个图片的标签**，然后让它们针对测试图片进行"投票"，最后把得票数最高的标签作为对测试图片标签的预测。所以当 K=1 的时候，它就退化成了 Nearest Neighbor 分类器。从下图可以直观地感受到，更高的 K 值可以使分类的效果更加平滑，使得分类器对于异常值的处理更好。

![KNN-2](/notes-images/KNN-2.png)

为什么会出现白色区域？这是由于邻近标签最高票数相同导致的，也就是有**多个标签对于这张图片的相似性是一样的**。**白色区域是分类模糊的区域，即图像与两个以上的分类标签绑定。**

根据上面的例子可以看出来，对于 KNN 分类器来说，K 值的选取是至关重要的，下面我们就来讨论 K 值如何确定的问题。

### 超参选取（如 K 值）

K-NN 分类器需要设定 K 值，那么选择哪个 K 值最为合适呢？简单来说，就是哪个值的表现最好我们就选择哪个。但是，这又引出了另一个问题，怎么判断哪个值表现最好呢？寻找这个 K 值的过程，我们称之为**调优**。特别注意的是：**决不能使用测试集调优**。当我们在设计机器学习算法的时候，应该把测试集看做非常珍贵的资源，不到最后一步，绝不使用它。使用测试集调优的危险在于：**我们在测试的时候的性能也许会非常好，但是实际部署后，性能可能会远远低于预期**。这种情况我们称为**算法对测试集过拟合**。测试集应该在算法已经完成后，用于检测算法的性能，如果在算法设计时就使用了，后面再去检测的时候就没有意义了。

> **测试集只能使用一次，即在训练完成后用于最终模型的评价的时候**。

实际操作中，我们调优的思路是：**从训练集中取出一部分数据专门用来调优，我们将这一部分数据称之为验证集（validation set）**。

## Nearest Neighbor 分类器的优劣

Nearest Neighbor 分类器的优点在于：**易于理解，实现简单，在某些特定情况下（比如数据维度较低时），可能是不错的选择。**

但在现实场景中 **NN 并不适用于图像分类领域**，主要是因为：

**1. 在测试时速度非常慢，不符合实际需求**

一个优秀的算法，往往是**在训练时需要花费大量时间，而在测试的时候花费的时间就相对少很多**，这是我们希望一个优秀的算法达到的一个标准。而 **NN 算法**是**不需要花费时间训练的**，因为 NN 的训练只是将训练集的数据存储起来，这就导致 NN 在测试的时候需要花费大量的时间去计算，这是一个不能忍受的缺点。

**2. 难以处理高维数据**

Nearest Neighbor 分类器在维度较低的情况下，可能是不错的选择。但在实际图像分类工作中，**图像都是高维度数据（它们通常包含很多像素），而高维度向量之间的距离通常都是反直觉的，NN 算法中计算距离的方法就不太适用（L1、L2 距离）**。

具体来说，图片的排布更像是一种颜色分布函数，或者说是基于背景的，而不是图片的语义信息。比如，狗的图片和青蛙的图片可能就非常接近，因为这两种标签中的大多数图片背景都是白色。从理想效果来说，我们肯定希望**同类图片能够聚集在一起**，而不会被背景等不相关因素干扰。为了达到这个目的，我们不能止步于原始像素比较，所以 NN 算法就不太适用了。

## 线性分类器

前面我们提到了 NN 和 K-NN 算法，它们具有很大的局限性。所以我们需要一种更为强大的方法来解决图像分类问题。该方法可以自然地延伸到神经网络和卷积神经网络上。这种方法主要有两部分组成：一个是**评分函数（score function）**，它是原始图像数据到类别分值的映射。另一个是**损失函数（loss function）**，它是用来量化预测分类标签的得分与真实标签之间一致性的。该方法可转化为一个最优化问题，**在最优化过程中，将通过更新评分函数的参数来最小化损失函数值。** 这就是线性分类。
