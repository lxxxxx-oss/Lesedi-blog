# Ajax

## 一、原生AJAX

### AJAX简介

AJAX全称**Asynchronous JavaScript and XML**，就是**异步**的**JS**和**XML**

通过AJAX可以在浏览器中向服务器发送异步请求，最大的优势就是：**无刷新获取数据**

AJAX不是新的编程语言，而是一种将现有标准组合在一起使用的新方式

### XML简介

- XML就是**可扩展标记语言**
- XML被用来**传输**和**存储数据**
- XML和HTML类似，不同的是**HTML**中都是**预定义标签**(a div p span这些标签都是预定义的)，而**XML**中没有预定义标签，**全部都是自定义标签**

但现在XML已经被更简洁方便的**JSON**取代了

## 二、HTTP协议

### 请求报文

这里重点是格式和参数

请求报文包括四部分

- **请求行**
  - 行里面主要包含**请求类型**：GET POST
  - 然后是包含请求的**路径URL**
  - 最后是 **HTTP协议**的**版本号**，也是包含在请求行里面的 如HTTP/1.1
- **请求头**
  - 主要记住请求头的格式： **名字：空格+内容**
  - 请求头包含的东西比较杂乱其中有 ：
  - **Host**： xxxx.com
  - **Cookie**： name=xxx
  - **Content-type**： application/x-www-from-urlencoded。这告诉服务器**请求体的类型**
  - .....
- **空行**
  - 虽然为空，但必须得有，不可或缺
- **请求体**
  - 请求体根据请求行来决定有没有内容
  - 如果是GET请求，请求体则为空。若是POST请求，则请求体可以不为空

### 响应报文

同样四个部分

- 行
  - 包含三个部分
  - **协议版本** HTTP/1.1
  - **响应状态码**  一般来说2开头的2xx 都表示OK
  - **响应状态字符串**  与响应状态码对应
- 头
  - 与请求头一样
- 空行
- 体
  - 是服务器主要的**响应结果**
  - 就是一个**HTML格式**的内容
  - 浏览器接收到结果后，就会对HTML进行解析并对页面进行渲染

## 三、AJAX的特点

- 优点
  - 可以**无需刷新页面**而与**服务器**进行**通信**（这也是AJAX最大的优势）
  - 允许我们通过用户事件（**鼠标点击事件**、**键盘事件**等等）来更新部分页面内容
- 缺点
  - 没有浏览历史，不能回退
  - 存在跨域问题（可以通过代码解决）
  - SEO不友好

## 四、一个node工具

**nodemon**

作用是在我们修改了node数据的时候，自动帮我们重启项目

```bash
npm install -g nodemon
```

注意：当安装这个工具后 **启动**node.js文件就不是用 node+文件名了，而是使用**nodemon+文件名**

## 五、同源策略

同源策略最早由Netscape（网景）提出，是浏览器的一种**安全策略**

同源意为：**协议**、**域名**、**端口号**，完全一致。三者皆一致即为满足同源策略

**违背同源策略**即是我们所说的**跨域**

而AJAX是默认满足同源策略的，所以我们的跨域操作需要代码来改变AJAX的默认设置

```js
//这行代码就是设置允许跨域
 response.setHeader('Access-Control-Allow-Origin', '*')
```
