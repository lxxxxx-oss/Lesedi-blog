# 一些零碎但重要的知识点

## ES6

### 箭头函数

#### 箭头函数和普通函数的区别

1.箭头函数比普通函数**更加简洁**

如果没有参数，就直接写一个空括号即可

如果只有一个参数，可以省去参数括号

如果有多个参数，用逗号分割

如果函数体的返回值只有一句，可以省略大括号

如果函数体不需要返回值，且只有一句话，可以给这个语句前面加一个void关键字。最常用的就是调用一个函数：    let fn = () => void doesNotReturn()

2.箭头函数没有自己的this 箭头函数不会创建自己的this,所以它没有自己的this,它只会在自己作用域的上一层继承this。所以**箭头函数中的this的指向在它在定义时就确定**了，之后不会改变。

3.箭头函数继承来的**this指向永远不会改变**

4.call()、apply()、bind()等方法不能改变箭头函数中的this指向

5.箭头函数**不能作为构造函数使用**

6.箭头函数**没有自己的arguments**

7.箭头函数**没有prototype**

8.箭头函数不能用作Generator函数,不能使用yield关键字

## CSS

### BFC

#### BFC是什么东西

BFC全称**Block Formatting Context**，名为"**块级格式化上下文**"

`BFC`是一个**完全独立的空间**（布局环境）、独立的渲染区域，让空间里的**子元素不会影响到外面的布局**。那么怎么使用`BFC`呢，`BFC`可以看做是一个`CSS`元素属性

#### 怎样触发BFC

这里简单列举几个触发`BFC`使用的`CSS`属性

- overflow: hidden
- display: inline-block
- position: absolute
- position: fixed
- display: table-cell
- display: flex

#### BFC规则

- 内部盒子会在**垂直方向**，一个接一个地放置。 Box垂直方向的距离由margin决定。

- 属于同一个BFC的两个相邻Box的margin会发生重叠。
- BFC的区域不会与float box重叠。
- BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。
- 计算BFC的高度时，浮动元素也参与计算。

## Vue

### Vue的生命周期

**Vue 实例有一个完整的生命周期**，也就是从开始创建、初始化数据、编译模版、挂载Dom -> 渲染、更新 -> 渲染、卸载 等一系列过程，称这是Vue的生命周期。

### 每个钩子做的事情

1、**beforeCreate（创建前）** ：数据观测和初始化事件还未开始，此时 data 的响应式追踪、event/watcher 都还没有被设置，也就是说不能访问到data、computed、watch、methods上的方法和数据。

 2、**created（创建后）** ：实例创建完成，实例上配置的 options 包括 data、computed、watch、methods 等都配置完成，但是此时渲染得节点还未挂载到 DOM，所以不能访问到 `$el` 属性。

 3、**beforeMount（挂载前）** ：在挂载开始之前被调用，相关的render函数首次被调用。实例已完成以下的配置：编译模板，把data里面的数据和模板生成html。此时还没有挂载html到页面上。

 4、**mounted（挂载后）** ：在el被新创建的 vm.$el 替换，并挂载到实例上去之后调用。实例已完成以下的配置：用上面编译好的html内容替换el属性指向的DOM对象。完成模板中的html渲染到html 页面中。此过程中进行ajax交互。

5、**beforeUpdate（更新前）** ：响应式数据更新时调用，此时虽然响应式数据更新了，但是对应的真实 DOM 还没有被渲染。

 6、**updated（更新后）**：在由于数据更改导致的虚拟DOM重新渲染和打补丁之后调用。此时 DOM 已经根据响应式数据的变化更新了。调用时，组件 DOM已经更新，所以可以执行依赖于DOM的操作。然而在大多数情况下，应该避免在此期间更改状态，因为这可能会导致更新无限循环。该钩子在服务器端渲染期间不被调用。

7、**beforeDestroy（销毁前）** ：实例销毁之前调用。这一步，实例仍然完全可用，`this` 仍能获取到实例

8、**destroyed（销毁后）** ：实例销毁后调用，调用后，Vue 实例指示的所有东西都会解绑定，所有的事件监听器会被移除，所有的子实例也会被销毁。该钩子在服务端渲染期间不被调用。

## 浏览器

### 重绘和回流

要理解重绘和回流，必须先了解一下**浏览器的工作原理**

- 浏览器会使用流式布局（Flow Based Layout）
- 浏览器会把**HTML**解析成**DOM**，把**CSS**解析成**CSSOM**，DOM和CSSOM就组成了**Render Tree**（渲染树）
- 有了**render tree**，浏览器就知道了所有节点的样式，然后计算它们在页面的大小和位置，最后把节点绘制到页面上
- 由于浏览器使用的是流式布局，对**render tree**的计算通常只需要遍历一次就可以完成。但table布局不包括在内（这里不展开）

页面**每渲染一次**必定有回流或重绘

- **回流**（**Reflow**）
  - 当**render tree**大小或者位置发生变化，或某些属性发生改变，浏览器重新渲染部分或全部文档的过程称之为回流
- 会导致回流的操作
  - 页面首次渲染
  - 浏览器窗口大小发生改变
  - 元素尺寸或位置发生改变
  - 元素内容发生改变（文字数量或图片大小）
  - 元素字体大小变化
  - 添加或删除**可见**的DOM元素
  - 激活CSS伪类（如 **:hover**）
  - 查询某些属性或调用某些方法

- **重绘**
  - **样式的改变**，比如大小和位置发生变化
  - 大小和位置发生改变则样式一定发生改变，而样式发生改变则不一定是大小和位置发生改变
  - 所以**回流是一定会触发重绘**的，而重绘则不一定触发回流
  - **回流比重绘的性能代价更高**

#### 减少、避免回流

- 避免使用**table布局**
- 避免多层**嵌套式设置样式**

- 放弃传统操作DOM的方式，基于vue/react开始数据影响视图的模式
- 样式集中改变
- 元素批量修改
- 分离读写操作
- **脱离文档流**

综上，ECMAScript和JavaScript之间的关系就是：**前者是后者的标准，后者是前者的实践**。通常情况下，这两者是等价的

### 闭包

简单来说，闭包就是可以在**内层函数中访问到其外层函数的作用域**。

#### 优点

- **延长变量的生命周期**，如果执行上下文不被销毁，这个变量就可以一直存在
- **形成私有作用域**，使得某些变量不受外部干扰
- 突破函数独立作用域，可以从外部拿到里面的变量

#### 缺点

- **内存泄漏**，因为绕过了垃圾回收机制，这个变量一直存在，过度使用闭包会导致内存占用过多

#### 应用场景

- return 一个引用父级变量的函数
- 函数作为参数
- 自调用函数
- 循环赋值
- **回调函数**
- **防抖节流**
- 函数柯里化

**没咋理解**

  - 动态意味着选取的元素是可以影响到文档的，而静态则是取出后就无法影响了

### 防抖和节流

#### 防抖

**触发高频事件后N秒内函数只会执行一次**，如果N秒内高频函数再次被触发，则重新计算时间

- 当事件触发时，响应的函数并不会立即触发，而是会等待一定的时间
- 当事件密集触发时，函数的触发会被**频繁的推迟**
- 只有等待了**一段时间没有事件触发**，才会真正的执行响应函数

##### 应用场景

- 输入框中频繁的输入内容，搜索或者提交信息
- 频繁的点击按钮，触发某个事件
- 监听浏览器滚动事件，完成某些特定操作
- 用户缩放浏览器的resize事件

**函数防抖(debounce)**

- **实现方式**：每次触发事件时设置一个延迟调用方法，并且取消之前的延时调用方法
- **缺点**：如果事件在规定的事件间隔内不断被触发，则调用方法会被不断延迟

```js
//防抖debounce代码：
function debounce(fn,delay) {
    let timeout = null; // 创建一个标记用来存放定时器的返回值
    return function (e) {
        // 每当用户输入的时候把前一个 setTimeout clear 掉
        clearTimeout(timeout); 
        // 然后又创建一个新的 setTimeout, 这样就能保证interval 间隔内如果时间持续触发，就不会执行 fn 函数
        timeout = setTimeout(() => {
            fn.apply(this, arguments);
        }, delay);
    };
}
// 处理函数
function handle() {
    console.log('防抖：', Math.random());
}
        
//滚动事件
window.addEventListener('scroll', debounce(handle,500));
```

#### 节流

- **函数节流(throttle)**：高频函数触发，但在N秒内只会执行一次，所以**节流会稀释函数的执行频率**
- 当事件触发时，会执行这个事件的响应函数
- 如果这个事件被频繁触发，那么节流函数会按照一定的频率来执行函数
- 不管在这个中间有多少次触发这个事件，执行函数的频繁总是固定的
- **常用场景：**1**.**window对象的resize、scroll事件（debounce防抖,throttle节流），2.文字输入、自动完成的keyup事件（debounce防抖），3.拖拽时的mousemove事件、鼠标不断点击触发（throttle节流）

##### 函数节流(throttle)

- 实现方式：每次触发事件时， 如果当前有等待执行的延迟函数，则直接return

```js
//节流throttle代码：
function throttle(fn,delay) {
    let canRun = true; // 通过闭包保存一个标记
    return function () {
         // 在函数开头判断标记是否为true，不为true则return
        if (!canRun) return;
         // 立即设置为false
        canRun = false;
        // 将外部传入的函数的执行放在setTimeout中
        setTimeout(() => { 
        // 最后在setTimeout执行完毕后再把标记设置为true(关键)表示可以执行下一次循环了。
        // 当定时器没有执行的时候标记永远是false，在开头被return掉
            fn.apply(this, arguments);
            canRun = true;
        }, delay);
    };
}
 
function sayHi(e) {
    console.log('节流：', e.target.innerWidth, e.target.innerHeight);
}
window.addEventListener('resize', throttle(sayHi,500));
```

函数防抖和节流都是**为了限制函数执行的频次**，以优化函数触发频率过高导致的响应速度跟不上触发频率，出现延迟，假死和卡顿现象

## 正则表达式

### 何为正则表达式

正则表达式其实起源于数学领域，后来被引入到了计算机领域。它是一个**为了字符串模式匹配，从而实现搜索和替换功能**，换句话说，就是帮我们**去匹配指定规则的字符串**的工具。是一种用来描述规则的表达式。它的底层原理也很简单，就是使用**状态机的思想进行模式匹配**。

### 正则表达式的组成

基本组成元素是：**字符**和**元字符**。**字符**就是**基础计算机字符编码**，通常指的就是正则表达式里面的数字和字母，而**元字符**也被称为特殊字符，通常指一些表示**特殊语义的字符**，如：^ | & 等。

### 单个字符

最简单的正则表达式就是由数字和字符组成的，没有特殊的语义，纯粹的一对一关系

```js
//例如这里需要匹配字母a，直接/a/即可
console.log(/a/.test('apple')) //true
```

稍微需要注意一点的是，在**匹配特殊语义的字符**的时候，需要**转义字符\\**来把特殊语义的字符转义为普通字符。

例如我想匹配`*`，首先用要用反斜杠`\*`让它失去原本的含义,然后再用单个字符匹配原则

```
/\*/
```

而如果这个字符原本不是特殊字符，使用反斜杠就会让它用有特殊的含义，如下表

| 特殊字符   | 正则表达式 | 记忆方式                                     |
| ---------- | ---------- | -------------------------------------------- |
| 换行符     | \n         | **n**ew line                                 |
| 换页符     | \f         | **f**orm feed                                |
| 回车符     | \r         | **r**eturn                                   |
| 空白符     | \s         | **s**pace                                    |
| 制表符     | \t         | **t**ab                                      |
| 垂直制表符 | \v         | **v**ertical tab                             |
| 回退符     | [\b]       | **b**ackspace,之所以使用[]符号是避免和\b重复 |

### 多个字符

单个的一对一的映射关系显然是不够的，所以需要引入**集合区间**和**通配符**的方式就可以实现一对多的匹配了。

在正则表达式中，集合定义的方式是使用中括号**[**  和  **]**。如**/[1234]/**这个表达式能匹配1，2，3，4这几个数字。当然这种方式数字少的时候还能用一用，需要匹配的数字如果比较多的话就显得有点低效。所以我们需要用到元字符 **-** 来表示区间范围。如**/[0-9]/**就表示能匹配所有数字，**/[a-zA-Z]/**则表示能匹配所有英文字母

正则表达式也给我们提供了一些常用的简便的匹配多个字符串的方式

|                   匹配区间                    | 正则表达式 |      记忆方式       |
| :-------------------------------------------: | :--------: | :-----------------: |
|         **除了换行符**之外的任何字符          |     .      | 句号,除了句子结束符 |
|                单个数字, [0-9]                |     \d     |      **d**igit      |
|                   除了[0-9]                   |     \D     |  **not** **d**igit  |
|    包括下划线在内的单个字符，[A-Za-z0-9_]     |     \w     |      **w**ord       |
|                  非单字字符                   |     \W     |  **not** **w**ord   |
| 匹配空白字符,包括空格、制表符、换页符和换行符 |     \s     |      **s**pace      |
|                匹配非空白字符                 |     \S     |  **not** **s**pace  |

**显然，大小写在这里表示的含义是恰恰相反的，可以借此来帮助记忆**

### 循环和往复

当然，匹配字符串的时候我们常常需要的是**匹配0、1、特定数次，甚至是无数次的字符**。所有一些表示循环往复的元字符就需要我们掌握

#### 元字符  ?

元字符**?**表示匹配一个字符的**0次或1次**。也就是说用到?的字符是**可有可无的，并且最多只能出现一次**

```js
//假如我想匹配 use 和 user 这两个单词
//那正则表达式应该为
/user?/
```

注意：**需要用到元字符的地方位于元字符的前面**，例如上面那个表达式，元字符?就对 r 发挥作用

#### 元字符  *

表示匹配一个字符的**0次或任意次**。也是用来**过滤可有可无的字符，不过不再限定它出现的次数**

#### 元字符  +

表示匹配的字符**至少要出现一次**

```js
//tab taab taaab
//匹配这三个字符串
/ta+b/
```

#### 特定次数

用中括号**{  }**就可以给重复匹配设置精确的范围。

```
- {x}: x次

- {min, max}： 介于min次到max次之间

- {min, }: 至少min次

- {0, max}： 至多max次
```

###  子表达式

前面讲的都是字符匹配，进阶的用法就需要用到子表达式了。通过**嵌套递归**和**自身引用**可以让正则发挥更强大的功能。

从简单到复杂的正则表达式通常需要采用**分组**、**回溯引用**和**逻辑处理**的思想。利用这三种规则，可以推演出无限复杂的正则表达式

#### 分组

所有被**（**和**）**元字符所包含的的正则表达式被称之为**一个分组**。每一个分组都是一个子表达式，它是构成高级和复杂正则表达式的基础

#### 回溯引用

回溯引用指的是模式**后面部分引用前面已经匹配到了的子字符串**

回溯引用的语法**\1,\2**... \1表示引用的一个匹配到的子表达式，\2就是第二个依此类推。比较特殊的是\0，它表示引用整个表达式

注意：回溯引用时**必须前面要有括号（）**括起来的表达式，第一个括号代表第一个子表达式。没有括号会报错

```
假设现在要在下面这个文本里匹配两个连续相同的单词，你要怎么做呢？
Hello what what is the first thing, and I am am scq000.
正则表达式：\b(what)\s\1
注意：整个正则表达式只匹配到了一个对象：what what。\b代表单词的开头，(what)代表这是第一个子表达式，\s代表匹配两个what之间的空格，没有它的话这个正则表达式就无匹配对象。\1代表回溯引用第一个表达式
```

回溯引用在**替换字符串**中十分常用，语法上有些许区别，用`$1`,`$2`...来引用要被替换的字符串。下面以js代码作演示：

```js
<!--利用回溯引用来替换字符串-->
let str = 'abc abc 123'
console.log(str.replace(/(a)bc/g, '$1g')); 
//括号后面的会被删除。相当于是括号里面的东西被引用了("$1")所以还存在，而括号后面的就直接被删除了。本来是都会被删除的
console.log(str.replace(/(abc)/g, '$1g'));  //abcg abcg 123
console.log(str.replace(/(a)/g, '$1g'));    //agbc agbc 123
console.log(str.replace(/(1)23/g, '$1g'));  //abc abc 1g

let num = '12345'
console.log(num.replace(/(12)/g,'$1,'))
```

如果想要**限制表达式的引用**，可以使用**非捕获正则**`(?:regex)`这样就可以避免浪费内存

```js
var str = 'scq000'
//没有使用非捕获正则
console.log(str.replace(/(scq000)/, '$1'));  //scq000
//使用非捕获正则
//由于使用了非捕获正则，所以$1没有值，这里直接替换为$1(替换成$1是replace的语法)
console.log(str.replace(/(?:scq000)/, '$1')); //$1
```

有时，我们需要**限制回溯引用**的适用**范围**。那么通过**前向查找**和**后向查找**就可以达到这个目的。

### 逻辑处理

逻辑与：在正则表达式里面，默认的正则逻辑就是**与**。所有逻辑与是没有对应的正则元字符的

逻辑非：**只有在`[`和`]`内部使用的`^`才表示非的关系**

| 逻辑关系 | 正则元字符  |
| -------- | ----------- |
| 与       | 无          |
| 非       | [^regex]和! |
| 或       | \|          |

### 浅拷贝和深拷贝

- 浅拷贝是创建一个新对象，这个对象有着原始对象属性值的一份精确拷贝。如果属性是基本类型，拷贝的就是基本类型的值，如果属性是引用类型，拷贝的就是内存地址 ，所以**如果其中一个对象改变了这个地址，就会影响到另一个对象**（**新对象会影响原始对象**）。
- 深拷贝是将一个对象从内存中完整的拷贝一份出来,从堆内存中开辟一个新的区域存放新对象,且**修改新对象不会影响原对象**。

总而言之，浅拷贝只复制指向某个对象的指针，而不复制对象本身，**新旧对象还是共享同一块内存**。但深拷贝会另外创造一个一模一样的对象，**新对象跟原对象不共享内存**，修改新对象不会改到原对象。
